\section{Escopo do cap\'itulo}
% ---------------------------------------------------------------

\begin{quote}\itshape
\textbf{Onde estamos na hist\'oria:} loop externo, fechando o ciclo.
Os cap\'itulos 11--14 instrumentaram, mediram e testaram o sistema.
Agora o loop externo entrega seu produto final: evid\^encias de qualidade
que alimentam o loop interno. Este cap\'itulo mapeia o caminho da maturidade
t\'ecnica e operacional --- de MVP a sistema de produ\c{c}\~ao sustent\'avel.
\end{quote}

Este cap\'itulo organiza o plano de evolu\c{c}\~ao do sistema em tr\^es
dimens\~oes: (1) \textbf{maturidade t\'ecnica} (do FAISS local ao OpenSearch
distribu\'ido); (2) \textbf{maturidade operacional} (do CI/CD manual ao
gate de qualidade autom\'atico); (3) \textbf{maturidade de avalia\c{c}\~ao}
(do ROUGE-L ao golden dataset com 50+ casos).

\section{Modelo de maturidade}
% ---------------------------------------------------------------

\subsection{N\'ivel 1 --- MVP Offline}

\begin{itemize}[leftmargin=*]
  \item FAISS local com documentos sint\'eticos;
  \item Agente ReAct com LangGraph e tool RAG;
  \item Avalia\c{c}\~ao manual por ROUGE-L offline;
  \item Deploy: notebook local sem AWS.
\end{itemize}

\textbf{Crit\'erio de sa\'ida:} 4/4 casos do golden dataset passam em modo mock.

\subsection{N\'ivel 2 --- Integra\c{c}\~ao AWS}

\begin{itemize}[leftmargin=*]
  \item Pipeline ass\'incrono: API Gateway $\to$ SQS $\to$ Lambda Worker
        $\to$ DynamoDB;
  \item OpenSearch substituindo FAISS em produ\c{c}\~ao;
  \item OTel + Langfuse para traces;
  \item CI/CD com gate de qualidade: ROUGE-L $\geq$ threshold por caso.
\end{itemize}

\textbf{Crit\'erio de sa\'ida:} p99 lat\^encia $< 30$\,s; zero regress\~oes
no golden dataset ap\'os cada deploy.

\subsection{N\'ivel 3 --- Produ\c{c}\~ao Sustent\'avel}

\begin{itemize}[leftmargin=*]
  \item Golden dataset $\geq$ 50 casos por segmento (PF/PJ/FP);
  \item LLM Judge calibrado com $\kappa \geq 0{,}60$;
  \item Dashboard CloudWatch com alarme para IC $< 0{,}40$ em mais
        de 10\% das respostas;
  \item Processo formal de expira\c{c}\~ao e atualiza\c{c}\~ao de
        casos do golden dataset.
\end{itemize}

\textbf{Crit\'erio de sa\'ida:} equipe de produto consegue atualizar
o golden dataset sem depend\^encia de engenharia.

\section{O ciclo de melhoria cont\'inua}
% ---------------------------------------------------------------

O modelo de evolu\c{c}\~ao deste sistema \'e circular, n\~ao linear:

\begin{enumerate}[leftmargin=*]
  \item \textbf{Loop externo produz evid\^encias:} IC baixo, regress\~ao
        no golden dataset, score do judge $< 7$.
  \item \textbf{An\'alise de causa raiz:} qual camada falhou? Prompt?
        RAG? Modelo? Segmenta\c{c}\~ao?
  \item \textbf{Loop interno recebe insumos:} ajuste de prompt,
        re-indexa\c{c}\~ao, atualiza\c{c}\~ao de pol\'iticas.
  \item \textbf{Deploy com gate de qualidade:} o c\'odigo s\'o avan\c{c}a
        se zero regress\~oes no golden dataset.
  \item \textbf{Loop externo valida:} IC sobe, judge aprova, usu\'ario
        satisfeito.
\end{enumerate}

Este ciclo transforma qualidade de software em qualidade de IA: n\~ao se
trata apenas de c\'odigo correto, mas de respostas corretas para clientes reais.

% ---------------------------------------------------------------
\section{Resumo}
% ---------------------------------------------------------------

\begin{itemize}[leftmargin=*]
  \item Tr\^es n\'iveis de maturidade: MVP offline, integra\c{c}\~ao AWS,
        produ\c{c}\~ao sustent\'avel.
  \item Gate CI/CD e golden dataset s\~ao as entregas de maior impacto
        por menor esfor\c{c}o.
  \item A calibra\c{c}\~ao do LLM Judge fecha o ciclo de confian\c{c}a
        na avalia\c{c}\~ao autom\'atica.
  \item O loop de melhoria cont\'inua transforma evid\^encias de produ\c{c}\~ao
        em melhorias no loop interno.
\end{itemize}
